# Комплексный анализ пользовательского поведения и кластеризация ресторанов для сервиса доставки еды

Данный проект представляет собой полноценное аналитическое исследование для сервиса доставки еды «Всё.из.кафе». Основная цель — выявить точки роста бизнеса: понять, как пользователи взаимодействуют с продуктом, где теряются на пути к заказу, как сегментировать аудиторию для повышения лояльности и как кластеризовать рестораны для персонализированных маркетинговых стратегий.

Проект выполнен в Jupyter Notebook и демонстрирует навыки работы с большими данными, построения воронок, RFM-анализа и машинного обучения (K-Means).

---

## 🎯 Задачи исследования

1. **Анализ пользовательского пути**  
   - Построение классической и step-by-step воронок для всех новых пользователей.  
   - Сравнение воронок для мобильной и десктопной версий приложения.  
   - Выявление этапов с наибольшей потерей пользователей.

2. **RFM-сегментация пользователей**  
   - Расчёт метрик: давность (Recency), частота (Frequency), сумма трат (Monetary).  
   - Построение гистограмм распределения каждой метрики.  
   - Выделение сегментов (R, F, M по 3 группам) и анализ распределения пользователей по сегментам.

3. **Кластеризация ресторанов**  
   - Подготовка данных: общее количество заказов, средний чек, число уникальных блюд.  
   - Определение оптимального числа кластеров методом «локтя» и иерархическая кластеризация (дендрограмма).  
   - Кластеризация методом K-Means, описание полученных кластеров, визуализация зависимостей между признаками.

4. **Формулировка рекомендаций** для улучшения конверсии, удержания пользователей и работы с партнёрами-ресторанами.

## 🔍 Основные шаги анализа

### 1. Загрузка и предобработка данных
- Данные загружены с корректным парсингом дат.
- Проверка типов, приведение целочисленных идентификаторов к `Int64`.
- Дубликаты отсутствуют. Пропуски объясняются логикой данных (например, поля заказа заполнены только для событий `order`).

### 2. Анализ воронок продаж
- Определены этапы пользовательского пути:  
  `authorization` → `main_page` → `reg_page` → `confirm_phone` → `object_page` → `add_to_cart` → `order`.
- Построены две воронки:
  - **Классическая** — доля пользователей от первого шага.
  - **Step‑by‑step** — конверсия между соседними шагами.
- Визуализированы обе воронки.
- Выполнено сравнение воронок для `Mobile` и `Desktop`.
- **Выявлены узкие места:**
  - На **десктопе** наибольшая потеря — с `main_page` на `reg_page` (переходят ~79%).
  - На **мобильных** устройствах — с `reg_page` на `confirm_phone` (переходят ~55%).

### 3. RFM-сегментация
- За дату анализа принято **3 июля 2021**.
- Для каждого пользователя с заказами рассчитаны:
  - **Recency** (дней с последней покупки)
  - **Frequency** (количество заказов)
  - **Monetary** (суммарная выручка)
- Построены гистограммы для каждой метрики.
- RFM-сегменты выделены по трём группам:
  - R: три равные группы по давности.
  - F: 1 заказ, 2 заказа, 3 и более.
  - M: по 33-му и 66-му перцентилям.
- Составлена таблица распределения пользователей по сегментам.
- **Ключевые наблюдения:**
  - Большинство пользователей делают 1–2 заказа, давность сильно варьируется.
  - Самые ценные сегменты (333, 332, 323, 233) имеют малую долю, но требуют особого внимания.
  - Многочисленные группы (211, 212, 311, 312, 111) нуждаются в акциях для повышения частоты и суммы заказов.

### 4. Кластеризация ресторанов (K‑Means)
- Для каждого ресторана рассчитаны признаки:
  - `orders_count` — общее число заказов
  - `revenue_mean` — средний чек
  - `unique_obj` — количество уникальных блюд в заказах
- Данные нормализованы с помощью `StandardScaler`.
- Методом «локтя» и дендрограммой определено оптимальное число кластеров = **4**.
- Проведена кластеризация K‑Means.
- Вычислены средние значения признаков для каждого кластера.
- Построены графики зависимостей: «средний чек vs количество заказов» и «уникальные блюда vs количество заказов».
- **Характеристики кластеров:**
  - **Кластер 1:** низкое количество заказов, низкий средний чек, умеренное разнообразие.
  - **Кластер 2:** очень высокое количество заказов (≈1100), широкий ассортимент, средний чек ниже, чем в кластере 4.
  - **Кластер 3:** средние показатели по всем параметрам.
  - **Кластер 4:** самый высокий средний чек (≈4657 руб.), но минимальное разнообразие блюд (2–3). Перспективен для расширения меню.

## 📈 Ключевые результаты и рекомендации

### 1. Проблемы конверсии на разных платформах
- **Desktop** — плохая заметность кнопки регистрации на главной странице.
- **Mobile** — сложный процесс подтверждения телефона при регистрации.
- **Рекомендации:** упростить мобильную регистрацию, улучшить визуальное выделение кнопки регистрации на десктопе.

### 2. RFM-сегменты
- Для массовых сегментов (211, 212, 311, 312, 111) предлагаются:
  - Накопительные бонусы за частоту заказов.
  - Промокоды на сумму заказа (стимуляция среднего чека).
  - Кэшбэк, возрастающий с суммой покупки.
- Для малых, но ценных сегментов (333, 332, 323, 233) — точечные персональные предложения.

### 3. Кластеры ресторанов
- **Кластер 1 и 3:** увеличить ассортимент для роста числа заказов.
- **Кластер 2:** поддерживать широкий выбор, работать над повышением среднего чека (например, кросс‑продажи).
- **Кластер 4:** расширять меню, но контролировать средний чек, чтобы не допустить его снижения.

### 4. Выявлена сильная корреляция между количеством уникальных блюд и числом заказов (≈0,996), что подтверждает важность разнообразия для привлечения клиентов.
