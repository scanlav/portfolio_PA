# Автоматизация отчетности: Daily ETL Pipeline для Яндекс Книг (Airflow & PySpark)

В данном проекте реализован автоматизированный процесс (ETL) ежедневной агрегации данных о пользовательской активности в сервисе Яндекс Книги. Проект демонстрирует навыки построения стабильных дата-пайплайнов и работы с Big Data стеком.

## 📋 Описание бизнес-задачи
Команде аналитиков требовался инструмент для быстрого получения данных по ежедневной активности пользователей. Ранее данные собирались вручную, что замедляло формирование продуктовых метрик. 

**Цель:** Автоматизировать сбор данных из S3, их обработку с помощью PySpark и загрузку агрегированных метрик в базу данных (ClickHouse) для последующей визуализации в BI.

## 🛠 Стек технологий
*   **Оркестрация:** Apache Airflow
*   **Обработка данных:** PySpark (Apache Spark)
*   **Инфраструктура:** Yandex Cloud (Dataproc, Object Storage)
*   **Хранение данных:** ClickHouse / СУБД с JDBC-интерфейсом
*   **Язык:** Python

## 📈 Структура решения

### 1. PySpark Job (`my_spark_job.py`)
Разработан скрипт для обработки больших объемов данных:
*   Чтение сырых CSV-файлов из облачного хранилища S3.
*   Фильтрация и агрегация данных на уровне пользователей (`puid`).
*   Расчет метрик: `audition_count` (количество сессий) и `avg_hours` (средняя длительность).
*   Запись результатов в аналитическую БД через JDBC.

### 2. Airflow DAG (`audition_content_analysis`)
Спроектирован направленный ациклический граф (DAG) со следующей логикой:
*   **S3KeySensor:** Мониторинг появления новых данных в бакете S3 за текущие сутки (динамическая подстановка даты через Jinja-шаблоны `{{ ds }}`).
*   **PysparkJobOperator:** Запуск вычислений на кластере Dataproc после успешного обнаружения файла.
*   **Обработка зависимостей:** Гарантированная последовательность выполнения задач и обработка ошибок.

## 📊 Результат
В результате работы пайплайна в базе данных ежедневно формируется витрина:

| puid | audition_count | avg_hours |
|------|--------------------|---------------|
| 68296740-f9d6 | 4 | 0,3675 |
| 6829675e-f9d6 | 1 | 1,3563 |

**Бизнес-эффект:**
1.  **Снижение Time-to-Insight:** Аналитики получают свежие данные к 16:00 ежедневно без ручного вмешательства.
2.  **Масштабируемость:** Использование Spark позволяет обрабатывать миллионы строк логов без потери производительности.
3.  **Надежность:** Автоматические повторы (retries) и сенсоры обеспечивают устойчивость процесса к задержкам в поставке данных.
