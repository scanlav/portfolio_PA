# Автоматизация продуктовой отчетности: ETL-пайплайн для данных такси (Airflow & PySpark)

Проект по созданию стабильного процесса сбора и агрегации данных о поездках такси. Решение позволяет перейти от ручного сбора данных к автоматизированным финансовым отчетам и продуктовым дашбордам.

## 📌 Бизнес-задача
Для оперативного мониторинга финансовых показателей (доход по типам оплаты, средние чеки, чаевые) требовалось создать систему, которая ежедневно:
1. Проверяет наличие свежих выгрузок от поставщиков данных.
2. Трансформирует и агрегирует сырые данные (Big Data).
3. Обновляет финальную таблицу в базе данных для визуализации.

## 🛠 Стек технологий
*   **Оркестрация:** Apache Airflow
*   **Обработка данных:** PySpark (Spark SQL/DataFrames)
*   **Инфраструктура:** Yandex Cloud (Dataproc, Object Storage)
*   **Хранение результатов:** ClickHouse (через JDBC)
*   **Инструменты:** Python, S3, SQL

## 📈 Структура проекта

### 1. Обработка данных (PySpark)
Разработан скрипт `my_spark_job.py`, который выполняет:
*   Чтение данных в формате Parquet из облачного хранилища.
*   **Агрегацию метрик** в разрезе `payment_type`:
    *   `taxi_count` — общее количество поездок.
    *   `avg_fare` — средняя стоимость поездки.
    *   `avg_tips` — средний размер чаевых (метрика лояльности/качества).
    *   `total_revenue` — суммарный оборот.
*   Обогащение данных временной меткой запуска для версионности отчетов.

### 2. Оркестрация (Airflow DAG)
Создан DAG `taxi_payment_summary`, реализующий логику пайплайна:
*   **S3KeySensor:** Ожидание появления файла в бакете перед запуском обработки (защита от неполных данных).
*   **DataprocCreatePysparkJobOperator:** Динамическое создание задачи в кластере Spark с передачей метаданных (timestamp запуска).
*   **Режим работы:** Ежедневный запуск (`@daily`) с обработкой инкрементальных данных.
